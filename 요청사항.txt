업체 요청사항

1. 디자인
   -> 디자인 부분은 마음대로. 주 사용 컬러는 #FFCD12(노랑)

* 변경 필요하다면 요청 후 변경 가능함. 자유롭게 의견 제시 가능합니다.

2. 핵심 기능

get_naver_rank_public (1회성 랭킹 체크 스크래핑 함수)

실행 시간 약 5초-6초 이내로 나왔으면 함.
* 현재 기준 총 실행시간 약 12초

< Issue >
접속한 IP 기준으로 24시간에 3회만 작동 가능함.
총 실행시간 12초 = 파이썬 코드 처리 시간 8초 + 네트워크 전송시간 3-4초

** 파이썬에서 플라스크로 보내는 네트워크 전송 시간을 단축하거나, 파이썬 코드 처리 시간을 단축해야함 **

-> ① 코드 개선 
        현재까지 찾아낸 방법은 동적 크롤링으로 원하는 페이지 이동 후, BeautifulSoup4로 HTML 소스 파싱하고
        파싱한 소스에서 필요한 부분만 정리해서 결과값 전달하는 방식이 제일 빠르지만 복잡함

-> ② 환경 개선
        네트워크 : 테스트 환경이 가정용 인터넷이라 통신이 느린것으로 의심. 빠른 회선 쓰는 가상PC 구매해서 테스트 필요
        PC 사양 : 현재 PC가 저사양이라 서버에 적합하지 않을수도 있음. 고민 필요함.


get_naver_rank (DB에 등록된 업체+키워드 순위 전체 체크 함수)

일 1회, 오후 2시에 실행, 120분 이내 전체 처리 목표

* 코드 테스트 및 편의상 현재는 1시간에 1번 (46분) 돌아가도록 설정됨.

< Issue >
네이버 정책 상 1분 이내 30회 초과 크롤링 요청 시, 약 5분~10분간 IP 막는 현상 있음.
-> ① 프록시 우회
        VPN 등으로 우회 접속해서 네이버의 블락 현상을 피하는 방법도 가능. 기능에 대한 구현이 필요함.

-> ② IP 라우팅 테이블
        IP를 구매한 후, 30개 데이터 실행시마다 IP를 변경하여 함수 실행. 기능에 대한 구현이 필요함.
